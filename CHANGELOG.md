# 更新日志

## v1.2.2 - 待发布

### 新增

- 新增总结模式，支持通用总结、课程笔记、会议纪要、测评结论和观点分析。
- CLI 新增 `--summary-mode` 参数，GUI 处理参数中新增“总结模式”下拉框。
- 转写前会从视频标题、作者、标签、分类、简介和本地文件名生成上下文提示，帮助 Whisper 识别专有名词。
- 总结阶段会参考来源上下文，降低专有名词被误听后继续误写到总结中的概率。

### 优化

- 优化长视频分块总结提示词，分块阶段更重视关键事实、跨块信息和去除跑题内容，合并阶段更重视去重和逻辑顺序。
- `--summary-only` 现在会真正重新生成总结，不再直接复用已有 `summary.md`。
- 示例配置中的默认 Whisper 模型与程序默认值保持一致。
- 上下文提示做了长度控制，分块总结只使用精简上下文，避免不必要的 token 消耗。

## v1.2.1 - 已发布

### 新增

- 新增常见失败场景的可操作提示，包括 Bilibili 412、API Key 缺失、ffmpeg 缺失、网络/代理问题和本地文件缺失。
- 新增中文发行说明，GitHub Release 会直接读取对应版本的说明文件。

### 优化

- 精简 README，保留快速开始、配置、输出、打包发行和常见问题。
- 更新说明改为中文，降低非开发用户阅读成本。

### 修复

- 移除 Qt 6 高 DPI 废弃设置，避免启动时出现 DeprecationWarning。
- 清理项目图标 PNG 的色彩配置，避免项目自带资源触发 libpng iCCP 警告。

## v1.2.0 - 已发布

- 新增正式应用图标资源，用于 GUI 和 Windows 发行包。
- 新增窗口标题与关于页版本号展示。
- 新增任务完成提示和任务栏提醒。
- 优化 GUI 整体观感，包括无边框标题栏、左侧导航、历史任务表格和扁平滚动条。
- 历史任务列表优先展示视频标题，并将完成状态拆成独立状态列。
- 发布版自动检测 ffmpeg、ffprobe 和 yt-dlp 的方式更稳定。
- Windows 下下载、转写和总结过程改为静默运行，不再弹出额外命令行窗口。
- GitHub Actions 打包环境固定为 `windows-2022`，并提前启用 Node.js 24 actions runtime。
